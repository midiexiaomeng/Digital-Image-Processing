# 最优二叉搜索树问题分析与算法实现

## 问题描述
最优二叉搜索树问题是指给定一组键和它们的搜索频率，构造一棵二叉搜索树，使得所有键的搜索代价最小。搜索代价定义为键的深度乘以搜索频率的总和。

## 算法原理
使用动态规划解决该问题：
1. 定义子问题：计算所有可能的子树[i..j]的最小搜索代价
2. 递推关系：对于每个子树[i..j]，尝试所有可能的根节点r，计算左右子树的代价加上总频率
3. 存储中间结果：使用二维数组存储子问题解

## 输入输出样例

### 样例1
输入：
keys = [10, 12, 20]
freq = [34, 8, 50]

输出：
最小搜索代价: 142
最优BST结构:
{
    'key': 20,
    'left': {
        'key': 10,
        'right': 12
    }
}

### 样例2
输入：
keys = [5, 10, 15, 20]
freq = [4, 2, 6, 3]

输出：
最小搜索代价: 37
最优BST结构:
{
    'key': 15,
    'left': {
        'key': 5,
        'right': 10
    },
    'right': 20
}

## 时间复杂度分析
1. 基本实现：O(n^3)
   - 三重循环：子树长度L、起始点i、根节点r
   - 每次计算需要O(n)时间求和频率
2. 优化空间：O(n^2)
   - 预处理频率前缀和数组
   - 将频率求和操作优化为O(1)

## 优化改进方法
1. Knuth优化：利用最优二叉搜索树的单调性，将时间复杂度降至O(n^2)
   - 观察到root[i][j-1] ≤ root[i][j] ≤ root[i+1][j]
   - 减少不必要的根节点尝试

2. 空间优化：
   - 使用滚动数组减少空间复杂度
   - 只存储必要的前一层计算结果

3. 实际应用优化：
   - 对于静态搜索树(不常更新)，可以预处理所有可能查询
   - 对于动态环境，考虑使用近似算法或平衡二叉搜索树

## 应用场景
1. 编译器设计中的关键字搜索
2. 数据库索引优化
3. 拼写检查器的字典存储
4. 任何需要高频搜索的静态数据集
