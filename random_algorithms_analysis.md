# 随机算法分析报告

## 1. 主元素问题

### 问题描述
在数组中找出出现次数超过一半的元素（主元素）。如果不存在则返回None。

### 算法实现
使用随机采样算法：
1. 随机选择数组中的一个元素作为候选
2. 统计该候选在数组中出现的次数
3. 如果次数超过一半则返回，否则重复采样

### 时间复杂度
- 最坏情况：O(kn)，其中k是采样次数，n是数组长度
- 期望情况：O(n)，因为主元素占比高时容易快速找到

### 输入输出样例
输入：[2, 2, 3, 2, 4, 2, 5, 2, 2]
输出：2

输入：[1, 2, 3, 4, 5]
输出：None

### 优化方法
1. 增加采样次数k可以提高正确率
2. 可以先排序再检查中间元素（确定性算法O(nlogn)）
3. Boyer-Moore投票算法（确定性O(n)算法）

## 2. 素数测试问题

### 问题描述
判断一个给定的正整数是否为素数。

### 算法实现
使用Miller-Rabin概率测试算法：
1. 将n-1表示为d×2^s
2. 随机选择基数a进行测试
3. 检查a^d ≡ 1 mod n或a^(d×2^r) ≡ -1 mod n
4. 重复k轮测试

### 时间复杂度
- 单轮测试：O(logn)
- 总体复杂度：O(klogn)

### 输入输出样例
输入：7
输出：True

输入：15
输出：False

输入：101
输出：True

### 优化方法
1. 对小素数使用确定性测试
2. 选择特定基数集合可以变成确定性测试（对于n < 2^64）
3. 预计算小素数表进行初步筛选
4. 使用AKS素数测试（确定性多项式时间算法）

## 综合比较

| 算法 | 类型 | 时间复杂度 | 空间复杂度 | 准确性 |
|------|------|------------|------------|--------|
| 主元素随机算法 | 概率 | O(kn) | O(1) | 可调节 |
| Miller-Rabin | 概率 | O(klogn) | O(1) | 可调节 |
| Boyer-Moore | 确定 | O(n) | O(1) | 100% |
| AKS | 确定 | O(log^6n) | O(1) | 100% |
