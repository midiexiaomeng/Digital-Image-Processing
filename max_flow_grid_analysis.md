# 方格取数问题分析与算法实现

## 问题分析

给定一个m×n的棋盘，每个方格包含一个正整数。要求选择若干方格中的数，满足：
1. 任意两个被选中的方格不能相邻（共享边）
2. 所选数的总和最大

这是一个典型的**最大权独立集**问题，可以转化为**二分图**上的网络流问题。

## 算法设计

### 网络流建模
1. 将棋盘方格按国际象棋棋盘方式黑白染色，形成二分图：
   - 黑色方格为左部节点
   - 白色方格为右部节点
2. 构建网络流图：
   - 源点S连接所有黑色方格，容量为方格数值
   - 所有白色方格连接汇点T，容量为方格数值
   - 每个黑色方格连接相邻白色方格，容量为∞

### 增广路算法实现
使用Edmonds-Karp算法（BFS寻找增广路）求解最大流：
```python
# 关键代码片段
def max_flow(self, s, t):
    flow = 0
    level = [-1]*self.size
    while True:
        self.bfs_level(s, t, level)
        if level[t] < 0:
            return flow
        iter_ = [0]*self.size
        while True:
            f = self.dfs_flow(s, t, float('inf'), iter_, level)
            if f == 0:
                break
            flow += f
        level = [-1]*self.size
```

## 输入输出样例

### 输入
```
3 3
1 2 3
3 2 3
2 3 1
```

### 输出
```
11
```

解释：选择(0,0)=1, (0,2)=3, (1,1)=2, (2,0)=2, (2,2)=1，总和为1+3+2+2+1=9 ❌  
更正：实际最大和为11（选择(0,2)=3, (1,0)=3, (1,2)=3, (2,1)=3）

## 时间复杂度分析

设棋盘大小为N=m×n：
1. BFS层次图构建：O(N)
2. DFS寻找增广路：O(N)
3. 最多进行O(N)次增广（因为每次至少增加1流量，最大流量为O(N)）

总时间复杂度：O(N²) = O(m²n²)

## 优化改进方法

1. **Dinic算法**：
   - 使用分层图和多路增广
   - 时间复杂度可优化到O(N^{1.5})

2. **当前弧优化**：
   - 记录每个节点已访问的边，避免重复检查
   - 实际运行时可减少DFS时间

3. **并行计算**：
   - 对大规模棋盘，可并行处理不同增广路

4. **启发式方法**：
   - 优先处理大容量边
   - 使用贪心策略初始化流
